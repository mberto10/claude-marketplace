# YouTrack Documentation Guide

Use YouTrack as the official stakeholder record. All content should be in German.

## Hard Rules

- Do not mention Linear IDs or Linear links in YouTrack.
- Use YouTrack ticket IDs only (e.g., AI-301).
- Keep updates concise and stakeholder-friendly.

## Epic Description Template

```markdown
# Projektziel
[1-2 Saetze: Was liefert das Projekt? Welcher Outcome?]

# Meilensteine
| Meilenstein | Ziel-KW | Status | Bemerkung |
|-------------|---------|--------|-----------|
| [M1] | KW XX | DONE/IN_PROGRESS/PLANNED/BLOCKED | [Optional] |
| [M2] | KW XX | DONE/IN_PROGRESS/PLANNED/BLOCKED | [Optional] |
```

## KW Comment Template

```markdown
## KW{XX}

**Updates:**
- [Update 1]
- [Update 2]

**Blocker:**
- [Blocker]   # or "Keine"

**Next Steps:**
- [Next step 1]
- [Next step 2]
```

## Status Icons

- DONE
- IN_PROGRESS
- PLANNED
- BLOCKED

## Quality Checklist

- Is the Projektziel outcome-oriented (not a task list)?
- Do milestones have Ziel-KW and status?
- Is the latest KW update from this week?
- Are blockers explicit and time-bound?
